01

Why it was built

The problem every team hits: tests that don’t keep up with code. ›

Every team ships faster than their tests can follow. New features land daily; test suites fall behind. Coverage gaps hide in the cracks. Flaky tests erode trust until the team stops reading CI results. Manual QE doesn’t scale, and bolting on yet another tool adds integration tax without solving the root cause.

Dragan Spiridonov built Agentic QE because existing solutions attacked one slice of the problem — generation or coverage or flakiness — but never the whole surface. A coordinated fleet of AI agents, each a specialist in one QE domain, can cover the full lifecycle: generate, analyze, execute, learn, and gate.

The project is open-source (MIT), free to use, fork, and contribute to. It is based on the Agentic QE Framework created by Dragan Spiridonov.

02

What problem it solves

Tests fall behind, coverage rots, flaky tests erode trust. ›

Modern codebases grow faster than any human team can test. Agentic QE attacks the full quality surface at once:

  • Test generation gap — writing unit, integration, property-based, and BDD tests by hand takes days. AQE generates them in seconds, across 15+ frameworks (Jest, Vitest, Playwright, Cypress, pytest, JUnit, Go, Rust, Swift, Flutter, and more).
  • Coverage blind spots — line-coverage tools tell you what ran, not what matters. AQE’s risk-weighted analysis identifies the most impactful untested code paths.
  • Flaky tests — tests that pass today and fail tomorrow. AQE uses ML-powered detection, root-cause analysis, and stabilization recommendations.
  • Pattern amnesia — every new test file starts from scratch. AQE remembers patterns across sessions and projects, getting better with every interaction.
  • Tool sprawl — one SAST tool here, a coverage tool there, a linter somewhere else. AQE coordinates 60 agents across 13 domains through one interface.
  • AI cost overhead — sending everything to the biggest model is expensive. AQE routes tasks to the right model tier automatically (Haiku for simple, Sonnet for moderate, Opus for critical).
03

Why now

Coding agents exist — now QE needs agents too. ›

Coding agents like Claude Code, Cursor, and Copilot accelerate development. Code output has increased by an order of magnitude for many teams. But quality engineering hasn’t kept pace — developers still write tests by hand, run coverage reports manually, and debug flaky CI one test at a time.

The MCP (Model Context Protocol) standard now lets AI tools share context and call specialized servers. AQE plugs into this: one aqe init --auto and your coding agent has 60 QE specialists on call. No new IDE, no new workflow. It meets developers where they already work.

And with local models like Ollama available for free, teams can now run routine test generation at $0 cost using AQE’s free-tier mode (AQE_FREE_TIER=1), falling back to cloud models only for complex reasoning.

04

How it works

A fleet of 60 specialized agents, coordinated by a Queen. ›

AQE organizes 60 agents into 13 domains. The Queen Coordinator decomposes any quality task, spawns the right domain specialists, coordinates them in parallel, and synthesizes results.

DomainAgentsWhat they do
Test Generation4Generate tests, TDD workflows, mutation testing, property testing
Test Execution3Run tests in parallel, handle retries, integration testing
Coverage Analysis2Find untested code, prioritize by risk
Quality Assessment4Go/no-go gates, risk scoring, adversarial review
Defect Intelligence4Predict bugs, find root causes, fix flaky tests
Requirements2Validate testability, generate BDD scenarios
Code Intelligence4Knowledge graphs, semantic search, change impact
Security3SAST/DAST, compliance audits, exploit validation
Contracts2API contracts, GraphQL schema testing
Visual & A11y3Visual regression, WCAG compliance, viewport testing
Chaos & Performance3Fault injection, load testing, performance validation
Learning4Cross-project learning, pattern discovery, metrics
Enterprise7SAP, SOAP, ESB, OData, JMS/AMQP/Kafka

Plus 7 TDD subagents (red, green, refactor, code/integration/performance/security reviewers) and the Fleet Commander for large-scale orchestration.

Intelligent Model Routing

TinyDancer routes tasks to the right model tier to minimize cost without sacrificing quality. Simple tasks (type additions, refactors) go to Haiku. Moderate tasks (bug fixes, test generation) go to Sonnet. Critical tasks (architecture, security, complex reasoning) go to Opus.

Pattern Learning

AQE learns from every interaction. Successful test patterns, coverage strategies, and defect indicators are stored and indexed. When generating tests for a new service, AQE searches for similar patterns from past sessions — even across different projects.

05

What solved looks like

Ask for tests and get them — generated, validated, gated. ›

After running aqe init --auto in your project, you can ask your coding agent for quality engineering tasks in plain English: 

"Generate tests for src/services/PaymentService.ts with 95% coverage target"

Output:

Generated 48 tests across 4 files
- unit/PaymentService.test.ts (32 unit tests)
- property/PaymentValidation.property.test.ts (8 property tests)
- integration/PaymentFlow.integration.test.ts (8 integration tests)
Coverage: 96.2%
Pattern reuse: 78% from learned patterns

Quality gates catch hollow tests automatically. Tautological assertions like expect(true).toBe(true) are rejected. Edge cases from historical patterns are injected into test generation. This is anti-sycophancy scoring — the system refuses to generate tests that pass without testing anything.

75 Skills, Trust-Tiered

Agents draw from a library of 75 skills, each rated by trust tier: 49 verified (full test suite), 7 validated, 5 structured, 5 advisory. Skills cover core testing, accessibility, security, chaos engineering, enterprise integration, and more.

06

Platform support

One MCP server, 11 coding agent platforms. ›

AQE works with 11 coding agent platforms through a single MCP server:

Claude Code GitHub Copilot Cursor Cline OpenCode AWS Kiro Kilo Code Roo Code OpenAI Codex CLI Windsurf Continue.dev 
# Set up all platforms at once
aqe init --auto --with-all-platforms

# Or add a platform later
aqe platform setup cursor
aqe platform list       # show install status
aqe platform verify cursor  # validate config

LLM Providers

AQE’s HybridRouter auto-detects available providers. Set one or more API keys and it picks the best route:

  • Claude (ANTHROPIC_API_KEY) — default
  • OpenAI (OPENAI_API_KEY)
  • Gemini (GOOGLE_AI_API_KEY) — free tier available
  • OpenRouter (OPENROUTER_API_KEY) — 300+ models
  • Ollama — local, free, offline
  • Azure OpenAI, Bedrock

Free local-model mode: export AQE_FREE_TIER=1 runs test generation on local Ollama at $0 cost, falling back to cloud only when needed.

How to start

Three commands and you’re running. ›

Quick Start

# Install globally from npm (public, free)
npm install -g agentic-qe

# Initialize your project (auto-detects tech stack, configures MCP)
cd your-project && aqe init --auto

# That's it — MCP tools are available immediately in Claude Code

All packages are public on npm: npmjs.com/package/agentic-qe

Alternative: Claude Code Plugin

For a lightweight install inside Claude Code only: 

# From a local checkout
git clone https://github.com/proffesor-for-testing/agentic-qe.git
claude --plugin-dir ./agentic-qe/plugins/agentic-qe-fleet

# Or from the marketplace
/plugin marketplace add proffesor-for-testing/agentic-qe
/plugin install agentic-qe-fleet

The plugin bundles 11 agents, 9 slash commands, 9 skills, and auto-registers the MCP server.

Development Setup

git clone https://github.com/proffesor-for-testing/agentic-qe.git
cd agentic-qe
npm install
npm run build
npm test -- --run

08

Use cases

From test generation to chaos engineering to security audits. ›
Test Generation

Generate a full test suite

Ask qe-test-architect to generate unit, integration, property-based, and BDD tests for any service file. Outputs framework-specific code (Jest, Vitest, Playwright, pytest, JUnit, Go, Rust, Swift, Flutter, etc.). 

"Use qe-test-architect to create tests for PaymentService with 95% coverage"
Coverage Analysis

Find coverage gaps by risk

qe-coverage-specialist finds untested code and ranks it by risk, not just line count. Critical paths get tested first. 

"Find coverage gaps in src/ and prioritize by risk"
Flaky Tests

Hunt and fix flaky tests

qe-flaky-hunter uses ML-powered detection to identify non-deterministic tests, provides root-cause analysis, and suggests stabilization fixes. 

"Use qe-flaky-hunter to stabilize flaky tests in tests/integration/"
Security

Coordinate a security audit

Combine qe-security-scanner (SAST/DAST), dependency scanning, API security, and chaos resilience testing in one coordinated sweep. 

"Run security scan on the authentication module"
TDD

Full Red-Green-Refactor cycle

qe-tdd-specialist coordinates 5 subagents: write failing tests, implement minimal code, refactor, code review, security review. 

"Use qe-tdd-specialist to implement UserAuth with full RED-GREEN-REFACTOR"
Chaos Engineering

Inject faults and test resilience

qe-chaos-engineer injects network partitions, latency spikes, and resource exhaustion to verify your system degrades gracefully. 

"Use qe-chaos-engineer to inject network partitions into order workflow"

Full Pipeline

The Queen Coordinator can run the entire quality pipeline in one shot: 

"Use qe-queen-coordinator to run full quality assessment:
1. Generate tests for src/services/*.ts
2. Analyze coverage gaps with risk scoring
3. Run security scan
4. Validate quality gate at 90% threshold
5. Provide deployment recommendation"
